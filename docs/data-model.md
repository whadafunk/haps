# Data Model

Drizzle ORM with PostgreSQL 16. All tables use UUID primary keys generated by
`gen_random_uuid()`. Timestamps are `timestamptz` (always UTC in DB, convert
to local timezone in application layer). Soft deletes use `deleted_at`; hard
deletes are only used for expired events and at user request.

---

## Core Tables (Phase 1)

### users

Admins, organizers, and registered members. Tier-1 and registered Tier-3 users.

```sql
users
  id            uuid        primary key default gen_random_uuid()
  email         text        not null unique
  password_hash text        not null                -- argon2id
  display_name  text        not null
  role          text        not null                -- 'admin' | 'organizer' | 'member'
  avatar_url    text
  subscribed    boolean     not null default false  -- stay-in-touch opt-in (Phase 2)
  created_at    timestamptz not null default now()
  updated_at    timestamptz not null default now()
```

### visitor_sessions

Cookie-based identity for non-registered visitors. Every browser gets one.

```sql
visitor_sessions
  id            uuid        primary key default gen_random_uuid()
  user_id       uuid        references users(id) on delete set null
                            -- null until session is claimed by registration
  display_name  text                                -- learned from first RSVP
  email         text                                -- optional, for recovery
  event_access  jsonb       not null default '{}'  -- {slug: 'attendee'|'editor'}
  created_at    timestamptz not null default now()
  last_seen_at  timestamptz not null default now()
```

### events

```sql
events
  id              uuid        primary key default gen_random_uuid()
  slug            text        not null unique       -- URL-safe, 8-12 chars
  organizer_id    uuid        not null references users(id)
  title           text        not null
  description     text                              -- markdown
  location        text
  starts_at       timestamptz not null
  ends_at         timestamptz
  timezone        text        not null              -- IANA tz string e.g. 'Europe/Bucharest'
  cover_image_url text
  theme           text                              -- theme key, see themes registry
  status          text        not null default 'draft'
                              -- 'draft' | 'published' | 'cancelled' | 'archived'
  show_guests     boolean     not null default true
  allow_comments  boolean     not null default true
  max_capacity    integer                           -- null = unlimited
  rsvp_deadline   timestamptz                       -- null = no deadline
  created_at      timestamptz not null default now()
  updated_at      timestamptz not null default now()
  expires_at      timestamptz                       -- null = no auto-expiry
```

### event_tokens

Edit and attendee tokens. Multiple attendee tokens per event are allowed
(e.g. "VIP", "general"). Only one edit token per event (enforced at app level).

```sql
event_tokens
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  type          text        not null                -- 'edit' | 'attendee'
  token_hash    text        not null                -- argon2id of raw token
  label         text                                -- 'general' | 'vip' | 'co-host' etc.
  revoked       boolean     not null default false
  created_at    timestamptz not null default now()
```

### rsvps

One RSVP per visitor session per event (or per user per event if registered).

```sql
rsvps
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  session_id    uuid        references visitor_sessions(id) on delete set null
  user_id       uuid        references users(id) on delete set null
  token_id      uuid        references event_tokens(id) on delete set null
                            -- which attendee token was used to gain access
  display_name  text        not null
  email         text                                -- optional; used for reminders
  status        text        not null                -- 'yes' | 'maybe' | 'no'
  head_count    integer     not null default 1
  note          text
  checked_in    boolean     not null default false
  checked_in_at timestamptz
  created_at    timestamptz not null default now()
  updated_at    timestamptz not null default now()

  -- one RSVP per session per event
  unique (event_id, session_id)
  -- one RSVP per registered user per event
  unique (event_id, user_id)
  -- at least one of session_id or user_id must be set (enforce in app layer)
```

### comments

```sql
comments
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  session_id    uuid        references visitor_sessions(id) on delete set null
  user_id       uuid        references users(id) on delete set null
  display_name  text        not null
  body          text        not null
  deleted_at    timestamptz                         -- soft delete
  created_at    timestamptz not null default now()
```

## Phase 2 Tables

### date_polls

```sql
date_polls
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  closes_at     timestamptz
  finalized_at  timestamptz                         -- set when host picks a winner
  winner_option_id uuid                             -- set when host picks
  created_at    timestamptz not null default now()

date_poll_options
  id            uuid        primary key default gen_random_uuid()
  poll_id       uuid        not null references date_polls(id) on delete cascade
  proposed_date date        not null
  proposed_time time
  sort_order    integer     not null default 0

date_poll_votes
  id            uuid        primary key default gen_random_uuid()
  option_id     uuid        not null references date_poll_options(id) on delete cascade
  session_id    uuid        references visitor_sessions(id) on delete set null
  user_id       uuid        references users(id) on delete set null
  created_at    timestamptz not null default now()
  unique (option_id, session_id)
  unique (option_id, user_id)
```

### custom_questions + rsvp_answers

```sql
custom_questions
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  question      text        not null
  type          text        not null                -- 'text' | 'select' | 'multiselect' | 'checkbox'
  options       jsonb                               -- array of strings for select/multiselect
  required      boolean     not null default false
  sort_order    integer     not null default 0

rsvp_answers
  id            uuid        primary key default gen_random_uuid()
  rsvp_id       uuid        not null references rsvps(id) on delete cascade
  question_id   uuid        not null references custom_questions(id) on delete cascade
  answer        text        not null
  unique (rsvp_id, question_id)
```

### event_templates

```sql
event_templates
  id              uuid        primary key default gen_random_uuid()
  organizer_id    uuid        not null references users(id) on delete cascade
  name            text        not null
  description     text
  default_theme   text
  default_description text
  custom_questions jsonb      -- snapshot of questions for this template
  created_at      timestamptz not null default now()
```

### magic_links (cross-device session recovery)

```sql
magic_links
  id            uuid        primary key default gen_random_uuid()
  user_id       uuid        not null references users(id) on delete cascade
  token_hash    text        not null                -- SHA-256 of raw token
  used          boolean     not null default false
  expires_at    timestamptz not null                -- 15 min TTL
  created_at    timestamptz not null default now()
```

## Phase 3 Tables

### media (memories wall)

```sql
media
  id            uuid        primary key default gen_random_uuid()
  event_id      uuid        not null references events(id) on delete cascade
  session_id    uuid        references visitor_sessions(id) on delete set null
  user_id       uuid        references users(id) on delete set null
  uploader_name text        not null
  url           text        not null                -- storage path or S3 URL
  type          text        not null                -- 'photo' | 'video'
  caption       text
  deleted_at    timestamptz
  created_at    timestamptz not null default now()
```

## Indexes (minimum required)

```sql
-- Fast event lookup (most common query)
create index on events (slug);
create index on events (organizer_id);
create index on events (status, starts_at);

-- RSVP lookups
create index on rsvps (event_id, status);
create index on rsvps (session_id);
create index on rsvps (user_id);

-- Comment feed
create index on comments (event_id, created_at) where deleted_at is null;

-- Session lookup (cookie auth)
create index on visitor_sessions (user_id) where user_id is not null;

-- Token validation
create index on event_tokens (event_id, type) where revoked = false;

-- Expiry job
create index on events (expires_at) where expires_at is not null;
```

## Naming Conventions

- Table names: `snake_case`, plural.
- Column names: `snake_case`.
- Foreign keys: `{referenced_table_singular}_id`.
- Enum-like columns use `text` with a check constraint or app-layer validation,
  not Postgres `ENUM` types (easier to alter with Drizzle migrations).
- Boolean columns: named as a positive assertion (`show_guests`, not
  `hide_guests`).
